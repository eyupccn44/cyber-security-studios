# Security Policy

## Reporting Security Issues

If you discover a security vulnerability in this project itself (not issues found using this tool), please report it responsibly.

**Do NOT open a public GitHub Issue for security vulnerabilities.**

Instead, please report via:
- GitHub Private Security Advisory (preferred)
- Email: eyupcecen22@gmail.com

We will respond within 48 hours and work to release a fix promptly.

## Scope of This Policy

This policy covers security issues **in the studio framework itself** — e.g., a hook script that could be exploited, a settings.json that grants unsafe permissions by default.

This policy does NOT cover:
- Vulnerabilities you found using this tool on third-party systems
- Questions about whether a technique is ethical/legal
- Issues with Claude Code itself (report to Anthropic)

## Ethical Use Reminder

This studio framework is intended exclusively for **authorized security research and professional engagements**. Any use for unauthorized access to systems is strictly prohibited and violates the LICENSE terms.
